## 特性

### 高级结果映射

resultMap的所有子元素：

| 元素          | 子元素      | 解释                                                  |
| ------------- | ----------- | ----------------------------------------------------- |
| id            |             | 一个 ID 结果;标记结果作为 ID 可以帮助提高整体效能     |
| result        |             | 注入到字段或 JavaBean 属性的普通结果                  |
| constructor   |             | 类在实例化时,用来注入结果到构造方法中                 |
|               | idArg       | ID 参数;标记结果作为 ID 可以帮助提高整体效能          |
|               | arg         | 注入到构造方法的一个普通结果                          |
| discriminator |             | 使用结果值来决定使用哪个结果映射                      |
|               | case        | 将鉴别器的列与case的value值比较，判断取哪一个结果映射 |
|               | association | 一个复杂的类型关联;许多结果将包成这种类型             |
|               | collection  | 复杂类型的集                                          |

#### 构造方法

Mybatis的构造函数注入功能允许我们在类初始化时就设置某些值，而不暴露其中的public方法。当我们使用数据库中的数据实例化一个对象且需要调用其构造函数时，就需要使用constructor 标签。

```java
public class Student {
    private Integer id;
    private String nameCn;
    private String nameEn;
    private String di;
    private Integer constructorId;

    public Student(Integer id) {
        this.id = id;
        this.constructorId = id;
        System.out.println("Constructor Executed. Constructed Id is " + constructorId);
    }
//Getters & Setters
}
```

```sql
-- 结果集映射
<resultMap id="studentResultMap" type="org.lucian.dto.Student">
	<constructor>
		<idArg column="id" javaType="java.lang.Integer"/>
	</constructor>
</resultMap>
```

```sql
-- 查询数据时会调用构造方法为属性赋值：先执行构造方法，为id和constructorId赋值，其它没有出现在结果集映射中的属性的值并没有被注入。
SELECT id,name_cn,name_en,di FROM student
注意：select语句中的列名如果与对象中的属性名完全相同，那么该列不需要出现在结果映射中也会被注入，例如上图的di属性。
```

#### 鉴别器

实现根据返回结果中某一列的值不同有着不同的结果集

- 结果映射写在鉴别器外部

```sql
-- 定义一个<discriminator>需要指定“column”和“javaType”。“column”用来帮助Mybatis寻找比对结果中的值，“javaType”用来保证类型上的匹配。具体的例子，如下：

<resultMap id="nameCnResult" type="org.lucian.dto.Student">
	<result property="nameCn" column="name_cn"/>
</resultMap>

<resultMap id="nameEnResult" type="org.lucian.dto.Student">
	<result property="nameEn" column="name_en"/>
</resultMap>
-- Mybatis将会从结果集合中得到每一条记录，并且比较其di的值。如果正确的匹配到了某个case的值，它就会使用对应的resultMap。具体来讲，如果di为D，则使用nameCn，如果di为I，则使用nameEn。
-- 如果没有任何值匹配到，才调用外部的结果映射。
-- 另外需要注意的是：鉴别器会自动调用无参的构造方法，所以如果Java Bean对象重载了带参数的构造方法，那么使用鉴别器时需要再手动添加一个无参的构造方法，否则会报错。
<resultMap id="studentResultMap" type="org.lucian.dto.Student">
	<id property="id" column="id"/>
	<result property="nameCn" column="name_cn"/>
	<result property="nameEn" column="name_en"/>
	<discriminator javaType="String" column="di">
		<case value="D" resultMap="nameCnResult"/>
		<case value="I" resultMap="nameEnResult"/>
	</discriminator>
</resultMap>
```

- 结果映射写在鉴别器内部

```java
//定义实体类
public Student() {
   System.out.println("Constructor Executed.");
}
```

```sql
-- 创建结果集映射
<resultMap id="studentResultMap" type="org.lucian.dto.Student">
	<id property="id" column="id"/>
	<result property="nameCn" column="name_cn"/>
	<result property="nameEn" column="name_en"/>
	<discriminator javaType="String" column="di">
		<case value="D" resultType="org.lucian.dto.Student">
			<result property="nameCn" column="name_cn"/>
		</case>
		<case value="I" resultType="org.lucian.dto.Student">
			<result property="nameEn" column="name_en"/>
		</case>
	</discriminator>
</resultMap>
```

#### 关联和集合

关联(association)元素用来处理数据模型中的“has-one”关系，而集合(collection)则用来处理“has-many”的关系。举例来说，比如我们说一篇博客，只有一个作者，它们是一对一的关系，而一片博客可能有很多条评论，它们是一对多的关系。

- 普通映射查询

```java
//创建一个BlogDto
public class BlogDto {
    private Integer authorId;
    private String authorName;
    private Integer blogId;
    private String blogTitle;
    private Integer commentId;
    private String commentText;
}
```

```sql
-- 查询时每条记录对应一个Dto：
SELECT A.id    AS authorId,
       A.name  AS authorName,
       B.id    AS blogId,
       B.title AS blogTitle,
       C.id    AS commentId,
       C.text  AS commentText
FROM AUTHOR A
         JOIN BLOG B ON A.id = B.fk_author_id
         JOIN COMMENT C ON B.id = C.fk_blog_id;
```

- 标准数据模型

```java
//Blog对象用Java类来描述数据模型
public class Blog {
    private Integer id;
    private String title;
    private Author author;
    private List<Comment> comments;
}
```

```sql
-- 利用关联和集合元素。定义resultMap
-- javaType指定对象中某个属性的类型,ofType指定对象的类型
-- collection标签里有一个全新的属性："ofType"，这个属性是必须的。它用来区分JavaBean（或者字段）属性类型和集合包含的类型。
-- association标签下的id和result会被注入到Author类的对象中然后注入Blog的author属性，collection标签下的id和result会被注入到Comment类的对象中然后加入一个ArrayList，注入Blog的comments属性。
<resultMap id="blogResultMap" type="org.lucian.dto.Blog">
	<id property="id" column="blogId"/>
	<result property="title" column="blogTitle"/>
	<association property="author" javaType="org.lucian.dto.Author">
		<id property="id" column="authorId"/>
		<result property="name" column="authorName"/>
	</association>
	<collection property="comments" ofType="org.lucian.dto.Comment">
		<id property="id" column="commentId"/>
		<result property="text" column="commentText"/>
		<result property="blogId" column="blogId"/>
	</collection>
</resultMap>
```

### 动态 SQL

#### choose, when, otherwise

```sql
-- 实现提供“id”就按“id”查找，提供“title”就按“title”查找，若都未提供，就返回所有符合条件的BLOG（where 1 = 1）
/*
若id=1，则符合第一个分支，choose标签返回第一分支下的SQL语句，其它分支都被舍弃：
若id为null，title=“title”，则符合第二条分支：
若id和title都为null，则返回otherwise下的SQL语句：
*/
<select id="selectDynamicSQL" resultType="java.lang.Integer" parameterType="java.util.Map">
    SELECT COUNT(*) FROM blog
    <choose>
        <when test="id != null and id != ''">
            WHERE id = #{id}
        </when>
        <when test="title != null and title !=''">
            WHERE title = #{title}
        </when>
        <otherwise>
            WHERE 1 = 1
        </otherwise>
    </choose>
</select>
```

#### trim, where, set

```sql
-- where标签
-- MyBatis 有一个简单的处理，使用where标签，它会自动判断AND或OR关键字前是否有其它条件语句，如果没有，则会自动将AND或OR关键字去除（只会智能去除，不会智能添加，因为无法判断是AND还是OR）。
<select id="selectDynamicSQL" resultType="java.lang.Integer" parameterType="java.util.Map">
    <where>
        <if test="id != null and id != ''">
            AND id = #{id}
        </if>
        <if test="title != null and title !=''">
            AND title = #{title}
        </if>
    </where>
</select>

-- trim标签
-- trim 元素也可以实现类似的功能。比如，和 where 元素等价的自定义 trim 元素为：
-- prefixOverrides 属性会忽略通过管道分隔的文本序列（注意此例中的空格也是必要的）。它带来的结果就是所有在 prefixOverrides 属性中指定的内容将被移除，并且插入 prefix 属性中指定的内容。
/*
若id=1，title=“title”，where标签会自动除去id前的AND关键字
若id为null，title=“title”，where标签会自动除去title前的AND关键字
若id和title都为null，则返回otherwise下的SQL语句
*/
<trim prefix="WHERE" prefixOverrides="AND |OR ">
    <if test="id != null and id != ''">
        AND id = #{id}
    </if>
    <if test="title != null and title !=''">
        AND title = #{title}
    </if>
</trim>
```
