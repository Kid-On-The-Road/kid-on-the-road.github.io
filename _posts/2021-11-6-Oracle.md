## docker下安装Oracle数据库

- 下载Oracle镜像

```bash
docker pull registry.cn-hangzhou.aliyuncs.com/helowin/oracle_11g
```

- 创建本地数据映射目录

```bash
mkdir -p -m 755 /Users/lin/Projects/data/oracle_home/oracle_data/{app,dpdump,oraInventory}
```

- 创建Oracle容器并运行

```bash
docker run -d -p 1521:1521 --name oracle_11g   -v  /Users/lin/Projects/data/oracle_home/oracle_data/app:/opt/oracle/app     -v  /Users/lin/Projects/data/oracle_home/oracle_data/dpdump:/opt/oracle/dpdump  -v  /Users/lin/Projects/data/oracle_home/oracle_data/oraInventory:/opt/oracle/oraInventory   registry.cn-hangzhou.aliyuncs.com/helowin/oracle_11g
```

- 进入容器

```bash
docker exec -it oracle_11g bash
```

- 配置一下环境变量，输入密码helowin(在此之前sqlplus命令不可用)

```bash
su root 
```

- 修改配置

```bash
vi /etc/profile 
```

```bash
//添加的配置
export ORACLE_HOME=/home/oracle/app/oracle/product/11.2.0/dbhome_2

export ORACLE_SID=helowin

export PATH=$ORACLE_HOME/bin:$PATH
```

- 使配置生效

```bash
source /etc/profile
```

- 切换为Oracle用户

```bash
su - oracle
```

- 登录sqlplus并修改sys、system用户密码

| 区别     | conn ./as sysdba                                     | conn /as sysdba                                       |
| -------- | ---------------------------------------------------- | ----------------------------------------------------- |
| 用户组   | 默认用户组为ora_dba组，不需要切换即可执行命令。      | 没有默认用户组，需要切换到ora_dba组下，才能执行命令。 |
| 系统验证 | 不需要进行操作系统验证，即可进行连接登录。           | 需要进行操作系统验证，才可进行连接登录成功            |
| 密码修改 | 允许在当前默认用户组路径下执行账号连接登录密码修改。 | 没有默认用户组，不能执行账号连接登录密码修改。        |

```sql
//运行sqlplus命令，进入sqlplus环境，不以任何用户登录
sqlplus /nolog
//以系统管理员身份连接数据库
conn /as sysdba
alter user system identified by oracle;
alter user sys identified by oracle;
ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;
exit 
```

- 查看Oracle实例状态

```bash
lsnrctl status
```

####  instantclient配置

- 复制配置

```bash
//进入Oracle容器
sudo docker exec -it myoracle11g /bin/bash

//进入tnsnames.ora目录
cd /home/oracle/app/oracle/product/11.2.0/dbhome_2/network/admin/

//编辑tnsnames.ora
vi tnsnames.ora

//复制文件中的内容
LISTENER_HELOWIN =
  (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))

HELOWIN =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = helowin)
    )
  )
```

### 用户操作

#### 创建删除修改

- 创建用户

```bash
//运行sqlplus命令，进入sqlplus环境，不以任何用户登录
sqlplus /nolog
//以系统管理员身份连接数据库
conn /as sysdba
```

- 创建用户并设置密码

```sql
create user username identified by password;
```

- 修改用户

```sql
alter user username identified by password;
```

- 删除用户

```sql
drop user username;
```

#### 授权

oracle提供三种标准角色（role）

| 角色名                | 解释                                                         |
| --------------------- | ------------------------------------------------------------ |
| connect(连接角色)     | 拥有Connect权限的用户只可以登录ORACLE(仅具有创建SESSION的权限)，不可以创建实体，不可以创建数据库结构。 |
| resource(资源角色)    | 是授予开发人员的。拥有Resource权限的用户只可以创建实体，不可以创建数据库结构。同时，当把ORACLE resource角色授予一个user的时候，不但会授予ORACLE resource角色本身的权限，而且还有unlimited tablespace权限，但是，当把resource授予一个role时，就不会授予unlimited tablespace权限。 |
| dba(数据库管理员角色) | 是授予数据库维护人员的。拥有全部特权，是系统最高权限，只有DBA才可以创建数据库结构。 |

- 授权

```bash
grant rolename, resource to username;
```

- 撤销授权

```bash
revoke rolename, resource from username;
```

- 查询权限

```sql
-- 查询对象权限：
SELECT * FROM dba_tab_privs a WHERE a.grantee = '用户名';
 
-- 查询角色权限：
SELECT * FROM dba_role_privs a WHERE a.GRANTEE = '用户名';

-- 查询系统权限：
SELECT * FROM dba_sys_privs a WHERE a.GRANTEE = '用户名';
```
